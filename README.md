<h1 align="center">IoT Device Fingerprinting and Anomaly Detection</h1>

<p align="center">
A Machine Learning pipeline for fingerprinting IoT device network behavior and detecting anomalous traffic generated by botnet-based attacks.
</p>

<hr>

<h2>Overview</h2>

<p>
Connected IoT devices, such as <b>smart doorbells</b> and <b>security cameras</b>, are frequently targeted by malware such as <b>Mirai</b> and <b>Gafgyt</b> botnets.
This project is a machine learning pipeline designed to fingerprint the normal network behavior of an IoT device and rapidly detect anomalous traffic patterns indicative of volumetric network attacks (e.g., <b>UDP Floods</b>).
</p>

<p>
The repository implements a highly optimized <b>XGBoost Classifier</b> that processes network statistical features and identifies malicious traffic with near-perfect precision and recall.
</p>

<hr>

<h2>Architecture</h2>

<p>The pipeline consists of three primary modules:</p>

<h3><code>preprocess.py</code> — Data Pipeline & Stratification</h3>

<ul>
    <li>Ingests pre-computed network feature datasets (115 statistical features per packet).</li>
    <li>Resolves Pandas memory fragmentation warnings via frame copying.</li>
    <li>Dynamically labels traffic:
        <ul>
            <li><b>0</b> → Benign</li>
            <li><b>1</b> → Attack</li>
        </ul>
    </li>
    <li>Performs a stratified <b>80/20 train-test split</b> to ensure class balance.</li>
    <li>Standardizes the feature space using <b>Scikit-Learn's StandardScaler</b> for optimal gradient boosting convergence.</li>
</ul>

<h3><code>train.py</code> — Model Training & Evaluation</h3>

<ul>
    <li>Initializes an <b>XGBClassifier</b> tuned for network anomaly detection.</li>
    <li>Evaluates the model against the test set.</li>
    <li>Outputs:
        <ul>
            <li>Confusion Matrix</li>
            <li>Classification Report</li>
            <li>Precision</li>
            <li>Recall</li>
            <li>F1-Score</li>
        </ul>
    </li>
    <li>Serializes and exports the trained model (<code>.pkl</code>) along with the scaler parameters for future deployment.</li>
</ul>

<h3><code>extractor.py</code> — Live Traffic Feature Extraction</h3>

<ul>
    <li>Parses raw <code>.pcap</code> network captures using <b>Scapy</b>.</li>
    <li>Aggregates packets into fixed, non-overlapping windows, computing statistical features (mean/std packet size, packet rate, mean inter-arrival time) that mirror the structure of the training dataset.</li>
    <li>Designed to run on freshly captured live traffic, enabling inference outside of pre-computed dataset features.</li>
</ul>

<h2>Dataset</h2>

<p>
This project uses the <b>N-BaIoT Dataset</b> (sourced from the <b>UCI Machine Learning Repository</b> / Kaggle).
The data represents real network traffic captured from a <b>Danmini Smart Doorbell</b>.
</p>

<h2>Installation & Setup</h2>

<h3>Clone the Repository</h3>

```bash
git clone <repository-url>
cd <repository-name>
```

<h3>Set Up a Virtual Environment (Recommended)</h3>

```bash
python -m venv venv
```

Activate it:

**Windows**

```bash
venv\Scripts\activate
```

**Linux / macOS**

```bash
source venv/bin/activate
```

<h3>Install Dependencies</h3>

```bash
pip install -r requirements.txt
```

<h3>Prepare the Dataset</h3>

Create a `data/` directory in the project root and place the required CSV files inside it.

```text
project/
│
├── data/
│   ├── 1.benign.csv
│   └── 1.mirai.udp.csv
│
├── preprocess.py
├── train.py
├── extractor.py
└── ...
```

<hr>

<h2>Usage</h2>

<h3>Train the Model</h3>

```bash
python train.py
```

<p>The script will:</p>

<ul>
    <li>Load and preprocess the dataset</li>
    <li>Train the XGBoost classifier</li>
    <li>Evaluate model performance</li>
    <li>Export the trained model and scaler</li>
</ul>

<h3>Extract Features from a Live Capture</h3>

```bash
python extractor.py
```

<p>
By default this runs a standalone check. To extract features from an actual <code>.pcap</code> file, import and call the function directly:
</p>

```python
from extractor import extract_live_features

features_df = extract_live_features("captures/traffic.pcap", window_size=100)
print(features_df)
```

<h2>Results & Insights</h2>

<p>
The model achieves an <b>F1-Score of 1.00 (100% accuracy)</b> on the test set.
</p>

<blockquote>
Although perfect accuracy often indicates overfitting in conventional machine learning tasks, it is expected in this scenario. Volumetric DDoS attacks such as Mirai UDP Floods generate an extremely distinct traffic pattern. A device that normally sends only a handful of packets per minute suddenly transmits thousands of nearly identical packets every second, creating a highly separable decision boundary for the XGBoost classifier.
</blockquote>
